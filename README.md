# fullstack-mern-email-survey-app

**_View this project:_** [https://fullstack-mern-email-survey-app.onrender.com/](https://fullstack-mern-email-survey-app.onrender.com/)

## Description

A fullstack application build with NodeJS, Express, React, Redux, and MongoDB. User login implemented with a Google OAuth authentication flow (Google+ API), facilitated by Passport JS. (CD/CI) Deployment on Render.

(Notes: remote MongoDB instance, Mongo model class, Mongo model instance, queries, integration with Mongoose library, integration of database with Oauth flow; configuration of sessions with Cookie Session NPM package/library, encryption and automatic expiration of cookie(s); serialization and deserialization of user and login/logout functionalities with Passport; attention to developer vs production environments (separate databases and deployments), environment variables, (server architecture:) set proxy server for development environment to forward routes from localhost:3000 to localhost: 5001; Concurrently package to efficiently run multiple servers (frontend and backend); Redux state management (concepts: store, reducers, action creators, action types, slices of state, redux-thunk middleware, integration with react-redux library); client-side routing with React Router vs server-side routing using Express (including deployment-specific routing issues arising from differences between development and production environments); completion of authentication on backend and frontend; client-side billing with Stripe API (test mode, handling of tokens, rendering of credits based on Redux state management of authenticated Mongoose user model); survey and mailer creation, integration with SendGrid API including clicktracking of users' votes (yes/no) in the email, route handling for user feedback, creation of subdocument collections with Mongoose, understanding of how memory limitations affect tradeoffs of structuring the data in different ways; Redux Form handling ...)
